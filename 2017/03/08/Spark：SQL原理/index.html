整体性：将SQL的查询操作和Spark程序无缝衔接。Spark SQL通过分布式数据集RDD提供了结构化的查询，并且Python, Scala, Java等多种语言的API。这种紧密的结合使得在利用SQL查询执行复杂的分析算法时变得简单

统一的数据访问：多源数据可以被装载和查询。Schema RDD提供了单一的接口，使得可以高效的处理包括Hive表、Parquet列式存储格式的文件以及JSON格式的文件在内的结构化数据

兼容Hive：在已有的数据仓库上，可以直接运行Hive的查询语句，Spark SQL重用了Hive的前端以及MetaStore。可以很好的兼容Hive已有的数据、查询和UDFs

标准的连接方式：Spark SQL提供server模式。可以通过JDBC以及ODBC等工业标准方法进行连接

可扩展性：在长查询和交互方面采用了同样的引擎。Spark SQL利用了RDD模型的优点，提供了mid-query的容错机制，可以支持large jobs而不用担心历史数据由于不同引擎导致的问题
